# AI Infra —— 体系结构 进程、线程及其通信

## 一、什么是进程（Process）

进程 = 程序的一次执行实例。

- 进程是操作系统进行**资源分配**的基本单位
- 每个进程拥有**独立的地址空间**（代码段、数据段、堆、栈）
- 进程之间相互隔离，一个进程崩溃不会影响其他进程

### 进程的内存空间

```
┌─────────────┐ 高地址
│   内核空间    │
├─────────────┤
│   栈 Stack   │  ← 局部变量、函数调用（向下增长 ↓）
│     ...      │
│   堆 Heap    │  ← malloc/new 动态分配（向上增长 ↑）
├─────────────┤
│   BSS段      │  ← 未初始化的全局变量
├─────────────┤
│   数据段      │  ← 已初始化的全局/静态变量
├─────────────┤
│   代码段      │  ← 程序指令（只读）
└─────────────┘ 低地址
```

## 二、什么是线程（Thread）

线程 = 进程中的一条执行流。

- 线程是操作系统进行**CPU调度**的基本单位
- 一个进程可以包含多个线程
- 同一进程内的线程**共享**进程的地址空间和资源

### 线程共享 vs 独有

- **共享**：代码段、数据段、堆、文件描述符、信号处理器
- **独有**：栈、寄存器、程序计数器（PC）、线程ID

> 💡 一句话理解：进程是"独立的房子"🏠，线程是"房子里的房间"🚪。房间共享水电（内存），房子之间要靠快递（IPC）通信。

## 三、进程 vs 线程 核心区别

| 对比维度 | 进程 Process | 线程 Thread |
|---------|-------------|------------|
| 定义 | 资源分配的基本单位 | CPU调度的基本单位 |
| 地址空间 | 独立 | 共享进程的 |
| 资源开销 | 重（创建/销毁慢） | 轻（共享资源） |
| 通信方式 | 需要IPC机制 | 直接读写共享内存 |
| 隔离性 | 强（互不影响） | 弱（可能连带崩溃） |
| 切换开销 | 大（需切换页表） | 小（同进程内切换） |

> 💡 选择建议：需要高隔离性 → 多进程（如分布式训练）；需要高性能通信 → 多线程（如异步推理）

## 四、进程间通信（IPC）

### 7种方式

1. **管道 Pipe** — 半双工，父子进程间，简单但只能单向
2. **命名管道 FIFO** — 无亲缘关系的进程也能用，以文件形式存在
3. **消息队列 Message Queue** — 内核维护的消息链表，可按类型读取
4. **共享内存 Shared Memory** ⭐ 最快 — 多个进程映射同一块物理内存，需配合同步机制
5. **信号量 Semaphore** — 用于进程同步，P操作（等待）V操作（释放）
6. **信号 Signal** — 异步通知机制，如 SIGKILL、SIGTERM
7. **Socket** — 支持不同机器间通信，网络编程基础

> 💡 AI Infra最常用：**共享内存 + Socket**。NCCL通信：同机用共享内存/NVLink，跨机用Socket/RDMA。

## 五、线程间同步方式

### 1. 互斥锁 Mutex
🔒 同一时刻只有一个线程能持有。加锁 → 访问共享资源 → 解锁。最基础的同步方式。

### 2. 读写锁 RWLock
📖 读共享，写独占。适合"读多写少"的场景。多个线程可同时读，写时必须独占。

### 3. 条件变量 Condition Variable
⏰ 线程等待某个条件成立后再继续。配合Mutex使用。经典场景：生产者-消费者模型。

### 4. 自旋锁 Spinlock
🔄 忙等待，不让出CPU。适合临界区非常短的场景。GPU编程中常见。

> 💡 面试常问：Mutex vs Spinlock？
> - Mutex：线程会睡眠，适合长临界区
> - Spinlock：一直循环等待，适合短临界区

## 六、死锁（Deadlock）

### 死锁的4个必要条件

1. **互斥条件** — 资源一次只能被一个进程使用
2. **持有并等待** — 持有资源的同时请求其他资源
3. **不可剥夺** — 已分配的资源不能被强行夺走
4. **循环等待** — 存在进程间的环形等待链

🔑 **打破任意一个条件即可避免死锁**

### 常见解决方案

- 破坏循环等待：对资源编号，按顺序申请
- 超时机制：等待超时自动释放
- 死锁检测：定期检查等待图是否有环
- 银行家算法：分配前预判是否安全

## 七、在 AI Infra 中的应用

### 多进程场景
- **PyTorch DDP**：每个GPU一个进程，通过NCCL通信
- **vLLM多Worker**：每个Worker是独立进程，负责一张GPU

### 多线程场景
- **vLLM AsyncLLMEngine**：多线程处理请求调度
- **CUDA Stream**：GPU上多个Stream可并行执行kernel

### 通信实例
- **NCCL All-Reduce**：同机用共享内存/NVLink，跨机用Socket/RDMA（InfiniBand）
- **MPI**：分布式训练常用的进程间通信框架
